# Forest Poker

Next.js app for tracking poker **buy-ins / buy-outs** in **whole Israeli shekels (NIS)**, with transparent ledgers and minimal settlement transfers after each game.

## Stack

- **Next.js** (App Router), **Tailwind CSS**, **next-intl** (`en` default, `he` + RTL)
- **Firebase Authentication** (email/password + Google) + session cookie (`__session`)
- **Neon Postgres** + **Drizzle ORM**

### Firebase project (CLI-created)

- **Project ID:** `forest-poker-web-a8k3`
- **Console:** [Firebase console](https://console.firebase.google.com/project/forest-poker-web-a8k3/overview)
- After setup, enable **Authentication** → **Sign-in method**: turn on **Email/Password** and **Google**.
- Under **Authentication** → **Settings** → **Authorized domains**, ensure **`localhost`** is listed (required for Google sign-in redirect back from `firebaseapp.com/__/auth/handler` during local dev).

Admin SDK: download the service account JSON from [Service accounts](https://console.firebase.google.com/project/forest-poker-web-a8k3/settings/serviceaccounts/adminsdk), save as `secrets/firebase-adminsdk.json`, and keep `GOOGLE_APPLICATION_CREDENTIALS=./secrets/firebase-adminsdk.json` in `.env`.

## Setup

1. Copy `.env.example` to `.env.local` and fill values.

2. **Neon**: create a database and paste `DATABASE_URL`.

3. Apply the schema:

   ```bash
   npm run db:push
   ```

   (Or use `npm run db:generate` + migrations if you prefer SQL migrations.)

4. **Firebase**: create a project, enable Email/Password and Google sign-in, add web app config → `NEXT_PUBLIC_FIREBASE_*`.

5. **Firebase Admin**: Project settings → Service accounts → generate key JSON → map `client_email` → `FIREBASE_CLIENT_EMAIL`, `private_key` → `FIREBASE_PRIVATE_KEY` (escape newlines as `\n` in `.env`).

6. Run locally:

   ```bash
   npm install
   npm run dev
   ```

   Open `http://localhost:3000/en/games` (middleware redirects `/` to `/en`).

## Deploy (Firebase App Hosting)

Connect this repo in the Firebase console, set secrets `DATABASE_URL`, Firebase Admin env vars, and `NEXT_PUBLIC_FIREBASE_*` for production. Follow [Firebase App Hosting](https://firebase.google.com/docs/app-hosting) to add an `apphosting.yaml` / backend config as generated by the Firebase CLI or console wizard.

**GitHub Actions** (`.github/workflows/deploy-firebase-main.yml`): add repository secret **`FIREBASE_SERVICE_ACCOUNT_JSON`** — paste the **entire** service account JSON (same file as local `secrets/firebase-adminsdk.json`). The workflow authenticates `firebase deploy` with **`GOOGLE_APPLICATION_CREDENTIALS`** instead of the deprecated **`FIREBASE_TOKEN`**. Ensure the account has permission to deploy (e.g. **Firebase Hosting Admin**, **Firebase Rules Admin**, and App Hosting–related roles as needed for your project).

If you **rename the GitHub repository**, open Firebase → App Hosting → your backend → **connected repository** and reconnect or verify the link so deploys still run.

**Google sign-in (production):**

1. Firebase Console → **Authentication** → **Settings** → **Authorized domains**: add your App Hosting URL host (for example `forest-poker--forest-poker-web-a8k3.us-east4.hosted.app`), **without** appending `:8080`.
2. Same section usually includes `localhost` and `<project-id>.firebaseapp.com`; keep **NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN** as `<project-id>.firebaseapp.com` unless you use a custom auth domain.
3. App Hosting env must include valid **FIREBASE_CLIENT_EMAIL** and **FIREBASE_PRIVATE_KEY** so `/api/auth/session` can mint the `__session` cookie after Google redirects back.
