# Forest Poker

Next.js app for tracking poker **buy-ins / buy-outs** in **whole Israeli shekels (NIS)**, with transparent ledgers and minimal settlement transfers after each game.

## Stack

- **Next.js** (App Router), **Tailwind CSS**, **next-intl** (`en` default, `he` + RTL)
- **Firebase Authentication** (email/password + Google) + session cookie (`__session`)
- **Neon Postgres** + **Drizzle ORM**

## Setup

1. Copy `.env.example` to `.env.local` and fill values.

2. **Neon**: create a database and paste `DATABASE_URL`.

3. Apply the schema:

   ```bash
   npm run db:push
   ```

   (Or use `npm run db:generate` + migrations if you prefer SQL migrations.)

4. **Firebase**: create a project, enable Email/Password and Google sign-in, add web app config → `NEXT_PUBLIC_FIREBASE_*`.

5. **Firebase Admin**: Project settings → Service accounts → generate key JSON → map `client_email` → `FIREBASE_CLIENT_EMAIL`, `private_key` → `FIREBASE_PRIVATE_KEY` (escape newlines as `\n` in `.env`).

6. Run locally:

   ```bash
   npm install
   npm run dev
   ```

   Open `http://localhost:3000/en/games` (middleware redirects `/` to `/en`).

## Deploy (Firebase App Hosting)

Connect this repo in the Firebase console, set secrets `DATABASE_URL`, Firebase Admin env vars, and `NEXT_PUBLIC_FIREBASE_*` for production. Follow [Firebase App Hosting](https://firebase.google.com/docs/app-hosting) to add an `apphosting.yaml` / backend config as generated by the Firebase CLI or console wizard.
